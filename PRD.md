# 产品需求文档（PRD）：家庭纯P2P局域网通讯软件

**版本**：1.0\
**目标用户**：家庭成员（包括老人、儿童）\
**设备规模**：≤30台（手机、平板、电脑）\
**核心架构**：绝对去中心化，无任何服务器（包括信令服务器），仅依赖局域网广播/组播

***

## 1. 项目背景与目标

### 1.1 背景

现有家庭内网传输文件或聊天，通常依赖QQ/微信等外网服务，速度受限于宽带，且存在隐私顾虑。纯粹的P2P局域网工具（如飞秋、Feem）要么功能单一（只有文件传输），要么界面陈旧且无移动端，要么不支持音视频通话。因此需要一款**面向家庭、开箱即用、功能完整**的纯P2P通讯软件。

### 1.2 目标

- 提供零配置的设备发现与群组管理
- 支持点对点文本聊天、文件/文件夹高速传输
- 支持群聊（需密码加入）
- 支持音视频通话（无服务器信令）
- 提供“面对面快传”功能（二维码或临时码）
- 所有通信端到端加密，数据永不离开局域网

***

## 2. 用户角色与场景

| 角色   | 典型场景                | 关键需求          |
| :--- | :------------------ | :------------ |
| 家长   | 给孩子的电脑发送大文件（电影、安装包） | 高速传输，断点续传     |
| 孩子   | 与兄弟姐妹聊天、分享照片        | 群聊、表情、图片      |
| 老人   | 与家人视频通话             | 一键呼叫，大字体，简单界面 |
| 所有成员 | 在客厅聚会时互传照片          | 面对面快传（扫码或输码）  |

## 3. 功能需求

### 3.1 设备发现与在线状态

- **自动发现**：软件启动后，自动通过UDP广播/组播发现局域网内所有运行本软件的设备，无需任何配置。
- **设备列表**：主界面显示所有在线设备的昵称、设备类型图标（手机/电脑）、在线/离线状态（离线显示灰色）。
- **设备信息**：每个设备拥有唯一ID（基于MAC+随机数生成），可自定义昵称（默认“品牌+型号”）。
- **心跳维护**：每30秒发送心跳，超过90秒未收到则标记离线。

### 3.2 点对点文本聊天

- 从设备列表选择任一在线设备，进入聊天窗口。
- 支持发送：纯文本、emoji表情、图片（缩略图+原图）、短语音（按住录制）。
- 消息显示发送时间、已送达状态（因纯P2P无服务器，仅显示“已送达对方设备”）。
- 聊天记录本地存储（SQLite），支持按关键字搜索。

### 3.3 文件/文件夹传输

- **发送方式**：拖拽文件/文件夹到聊天窗口，或点击“发送文件”按钮。
- **支持特性**：
  - 多文件并发传输
  - 文件夹保留目录结构
  - 断点续传（传输中断后，下次对方在线可继续）
  - 传输进度条、速度显示（MB/s）
  - 完成后通知接收方
- **限制**：单个文件最大支持4GB（FAT32兼容性，实际可更大）。

### 3.4 群聊（带密码校验）

- **创建群组**：任意成员可创建群组，生成**群ID**（随机UUID）并设置**加入密码**（4-8位数字或字母）。
- **加入群组**：其他成员在“加入群组”界面输入群ID和密码，验证通过后加入。
- **群组管理**：
  - 创建者可修改密码、踢出成员（但成员退出后仍可重新加入，除非修改密码）。
  - 成员可主动退出群组。
- **消息分发**：由于无中心服务器，群消息采用**发送方点对点转发**机制：
  - 发送方维护本地的群成员列表（每个成员在线状态）。
  - 发送一条群消息时，遍历所有在线成员，分别通过TCP连接发送消息副本。
  - 离线成员的消息暂不存储（纯P2P无离线存储），除非用户手动勾选“等待离线成员上线后再发”（此时发送方本地队列暂存，待成员上线后自动发送）。
- **群聊界面**：显示群名称（可修改）、成员列表（在线/离线），支持文本、图片、文件。

### 3.5 音视频通话（无服务器信令）

- **发起通话**：从设备列表或群成员列表中选择一人，点击“语音通话”或“视频通话”。
- **信令机制**：利用UDP广播/组播交换SDP offer/answer和ICE候选。
  - 呼叫方生成session\_id，通过UDP广播发送`call_request`（包含SDP offer）。
  - 被呼叫方收到后弹出来电界面，接听后通过**单播UDP**回复`call_response`（包含SDP answer）。
  - 双方建立WebRTC PeerConnection，音视频流直连（host候选，无需STUN/TURN）。
- **通话界面**：
  - 显示对方视频（可选），自己的小窗。
  - 静音、扬声器、挂断按钮。
  - 支持切换前后摄像头（移动端）。
- **可靠性**：若广播信令丢包，呼叫方每200ms重发3次。超时则提示“对方无响应”。

### 3.6 面对面快传

- **场景**：两人在同一房间，想快速发送文件而不需在设备列表中查找。
- **方式一（二维码）**：
  - 发送方选择文件，点击“生成二维码”，界面显示二维码（包含临时会话ID、本机IP、端口）。
  - 接收方点击“扫一扫”，扫描后自动建立TCP连接，开始传输。
- **方式二（4位数字码）**：
  - 发送方选择文件，点击“生成临时码”，显示4位数字。
  - 接收方输入该数字码，软件自动广播查找发送方，建立连接后传输。
- **安全性**：临时会话一次性有效，传输完成即销毁。

### 3.7 设置与个性化

- **个人资料**：修改昵称、头像（从预设图标选择或拍照）。
- **网络**：手动指定UDP广播地址（如`255.255.255.255`）或组播地址（`239.0.0.1`），端口范围可配置。
- **存储**：更改文件保存路径，清理聊天记录。
- **加密**：显示设备公钥指纹，支持验证对方身份（可选高级功能）。
- **外观**：深色/浅色模式，字体大小调整（适合老人）。

## 4. 非功能需求

### 4.1 性能

- 内存占用：空闲<30MB，文件传输<100MB，视频通话<200MB。
- CPU占用：空闲<1%，传输时<10%，视频编码时<30%（取决于设备）。
- 传输速度：应接近局域网理论带宽（千兆网下>100MB/s）。

### 4.2 安全性

- **端到端加密**：所有文本、文件、音视频流均使用`libsodium`的`crypto_box`（Curve25519 + XSalsa20-Poly1305）或类似算法。公钥在设备首次发现时交换。
- **防伪造**：每条消息附带发送方签名，接收方验证。
- **群聊密钥**：群组拥有独立对称密钥（由创建者生成，并通过点对点加密分发给每个成员，成员输入密码后获取）。
- **无外网请求**：软件不得向任何外网IP发送数据（包括DNS查询），仅限局域网通信。

### 4.3 可用性

- **零配置**：安装后打开即用，无需注册、登录、输入IP。
- **界面友好**：大按钮、清晰图标、支持触摸和鼠标。
- **跨平台**：Windows 10/11、macOS 11+、Linux（Ubuntu 20.04+）、iOS 13+、Android 8+。
- **多语言**：简体中文、英文（后续可扩展）。

### 4.4 兼容性

- 兼容IPv4（优先），不强制要求IPv6。
- 在复杂网络（如子网隔离、VLAN）中，若广播不可达，提供手动输入目标IP地址进行连接的功能。

## 5. 技术架构要点（供trae开发参考）

| 模块      | 技术方案                                       | 说明                                  |
| :------ | :----------------------------------------- | :---------------------------------- |
| 跨平台框架   | **Flutter** (Dart)                         | 一套代码覆盖移动端+桌面端，UI一致性好                |
| 设备发现    | UDP广播 (端口8877) + mDNS备选                    | 使用`dart:io`的`RawDatagramSocket`     |
| TCP通信   | `dart:io`的`Socket`                         | 异步，自定义帧格式（长度前缀+JSON或Protobuf）       |
| 加密      | `cryptography` Dart包或Rust层                 | 使用`libsodium`的Dart绑定（`sodium_libs`） |
| 文件传输    | 多线程分块 + 断点续传                               | 每个文件块有偏移量，可复用TCP连接                  |
| 音视频通话   | `flutter_webrtc` + 自定义广播信令                 | 信令通过UDP广播/单播交换SDP和ICE               |
| 本地存储    | `sqflite` (SQLite)                         | 存储聊天记录、群组信息、设备列表                    |
| 二维码/条形码 | `mobile_scanner` (移动端) + `qr_flutter` (生成) | 桌面端可使用`qr_code`库生成图片                |

**关键实现细节**：

- **群消息分发**：发送方遍历群成员列表，为每个在线成员建立TCP连接（或复用已有连接），发送相同消息。为提高效率，可对同一设备上的多个群成员合并（但家庭场景一般每人一台设备）。
- **群组密码校验**：创建群时，生成群密钥（随机32字节）。用户输入正确密码后，创建者将群密钥通过点对点加密发送给加入者。后续群消息使用该群密钥加密。
- **离线消息队列**：发送方可选择“等待离线成员上线后自动发送”，此时本地存储待发消息，当检测到成员上线时立即发送。该队列大小限制100条/成员。

## 7. 验收标准

- 在家庭路由器环境下（3-5台设备），所有功能正常，设备发现时间<5秒。
- 文件传输速度不低于100Mbps（千兆内网）。
- 音视频通话延迟<200ms，无明显卡顿或花屏。
- 无任何外网流量（可用Wireshark验证）。
- 群聊密码错误无法加入，群消息仅群成员可解密。
- 离线设备重新上线后，能收到发送方队列中的消息（若开启等待功能）。

## 8. 风险与应对

| 风险                   | 应对措施                                                        |
| :------------------- | :---------------------------------------------------------- |
| 广播跨子网不可达             | 提供手动输入IP地址添加设备的功能                                           |
| WebRTC在部分设备上无音视频设备权限 | 提示用户授予麦克风/摄像头权限，并给出降级方案（仅文本）                                |
| 群消息点对点分发导致发送方带宽压力    | 限制群成员≤20人，每个消息大小≤10MB；文件仍建议点对点发送                            |
| 桌面端Flutter性能问题       | 使用`dart:ffi`调用原生库（如`libsodium`），音视频部分使用`flutter_webrtc`原生插件 |
